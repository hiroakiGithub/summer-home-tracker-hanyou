# 宿題トラッカー セットアップ手順書

所要時間：約10分。パソコンでの作業を推奨します（スマホでも可能）。

## 手順1：スプレッドシートを作る

1. Google Driveで新しいスプレッドシートを作成（名前は自由。例：「宿題データ」）
2. 下のシートタブを2つ用意し、名前をそれぞれ **config** と **data** にする
   - 最初からある「シート1」をダブルクリックして「config」にリネーム
   - 左下の「＋」で2枚目を追加し「data」にリネーム

## 手順2：configシートに宿題を入力する

1行目は見出し行です。**2行目から**宿題を入力します。

| 教科 | タイトル | 数値 | 単位 | 提出日 | 備考 |
|------|---------|------|------|--------|------|
| 国語 | 夏トレ | 16 | 回 | 8/25 | |
| 国語 | 読書感想文 | | | 8/3 | 原稿用紙3枚 |
| 数学 | サマースクール | 15 | 回 | 8/25 | |
| 理科 | 自由研究 | | | 始業式 | |
| 英語 | 毎日5分の音読 | | 毎日 | | 声に出して読む |
| その他 | ど根性ノート | 30 | ページ | 始業式 | |

**入力ルール：**
- **単位に「毎日」と入力** → 毎日型（その日にやったらチェックを入れるタイプ）になります。数値は空欄でOK。音読・日記・自主学習など、終わりが決まっていない毎日の習慣に使ってください
- **数値が2以上** → マス目式（第1回、第2回…と1つずつ記録するタイプ）になります
- **数値が空欄（または1）** → 「未着手／進行中／完了」で管理する単発課題になります
- **単位** → 「回」「ページ」「枚」など。空欄なら「回」になります
- **提出日** → 「8/3」の形式で書くと残り日数が自動計算されます。「始業式」などの文字でもOK（その場合カウントダウンなし）
- **提出日のセルは文字列で入力**してください（日付形式になってしまう場合は先頭に ' を付ける：'8/3）
- ⚠️ **一度使い始めたら、行の順番を変えたり途中に行を挿入したりしないでください。** 記録が別の宿題とずれます。追加する場合は一番下の行に足してください。

## 手順3：Apps Scriptを設定する

1. スプレッドシート上部メニュー「拡張機能」→「Apps Script」
2. 最初から書かれているコードを全て消し、以下を貼り付ける：

```javascript
function doGet(e) {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var cfg = ss.getSheetByName('config').getDataRange().getValues();
  var json = ss.getSheetByName('data').getRange('A1').getValue() || '{}';
  var out = { config: cfg, state: JSON.parse(json) };
  return ContentService.createTextOutput(JSON.stringify(out))
    .setMimeType(ContentService.MimeType.JSON);
}

function doPost(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('data');
  sheet.getRange('A1').setValue(e.postData.contents);
  return ContentService.createTextOutput(JSON.stringify({status:'ok'}))
    .setMimeType(ContentService.MimeType.JSON);
}
```

3. 上部の保存アイコン（フロッピー）をクリック

## 手順4：ウェブアプリとしてデプロイする

1. 右上「デプロイ」→「新しいデプロイ」
2. 左の歯車アイコン→「ウェブアプリ」を選択
3. 設定：
   - 実行するユーザー：**自分**
   - アクセスできるユーザー：**全員**（重要。「Googleアカウントを持つ全員」ではなく「全員」）
4. 「デプロイ」をクリック
5. 「アクセスを承認」→ 自分のGoogleアカウントを選択
6. 「このアプリはGoogleで確認されていません」という警告が出たら：
   - 「詳細」（Advanced）をクリック
   - 「（プロジェクト名）に移動」をクリック
   - 「許可」をクリック
   - ※自分で作った中身のわかっているスクリプトなので進めて大丈夫です
7. 発行された **ウェブアプリのURL**（https://script.google.com/macros/s/xxxxx/exec）をコピー

## 手順5：アプリに接続する

1. 共有されたアプリのURL（https://○○○.github.io/…）をiPadやスマホで開く
2. 最初の画面で、子どもの**名前**と、手順4でコピーした**URL**を入力して「追加して始める」
3. configシートに入力した宿題が表示されれば完了です

### きょうだいがいる場合（メンバー切り替え）

- 子ども1人につき、スプレッドシート1つが必要です。手順1〜4を人数分繰り返してください（スプレッドシートは「コピーを作成」で複製し、configの中身だけ書き換えると早いです。※コピーした場合もデプロイは各シートで新規に行う必要があります）
- アプリの画面上部「メンバーを切り替える」→ 名前とURLを入力して追加
- 以後、アプリを開くと名前の選択画面が表示され、タップでその子の記録に切り替わります
- メンバーが1人だけの場合は選択画面をスキップして直接開きます

※接続情報は端末ごとに保存されます。家族の各端末で同じ登録を一度ずつ行ってください。

## 注意事項

- **URLを知っている人は誰でも読み書きできます。** 家族・信頼できる範囲でのみURLを共有してください
- **同時に保存すると後の保存が優先**されます（先の保存が消えることがあります）。家族で同時刻の入力は避けてください
- configシートの宿題を後から追加した場合、アプリをリロードすれば反映されます

## 困ったとき

- 「接続できませんでした」→ デプロイ設定の「アクセスできるユーザー：全員」とシート名（config / data）を確認
- コードを修正したら → 「デプロイ」→「デプロイを管理」→ 鉛筆アイコン →「新バージョン」で再デプロイ（これをしないと反映されません）
- 保存したのに反映されない → dataシートのA1セルを確認。長い文字列が入っていれば保存はできています
